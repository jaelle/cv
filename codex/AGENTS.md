# Agents

## Planner
- Mandate: Keep the CV site single-source and simple; ensure inputs/outputs remain stable.
- Tools: none
- Inputs: /codex/README.md
- Outputs: Short task list for updates (e.g., "add new BibTeX entry; re-run build")
- Guardrails: No build complexity creep; avoid adding frameworks/themes.
- Escalates to: Builder

## Builder
- Mandate: Wire or adjust the GitHub Action; update Pandoc flags as needed.
- Tools: Git repo, GitHub Actions, Pandoc/PyPandoc
- Inputs: Planner tasks, `cv.md`, `publications.bib`
- Outputs: PRs modifying `build.yml`, `docs/index.html` (generated by CI)
- Guardrails: Never commit generated HTML manually; CI owns `docs/index.html`.
- Escalates to: Reviewer

## Reviewer
- Mandate: Verify output renders properly and citations resolve.
- Tools: Repo read, Action logs, HTML preview
- Inputs: PRs, CI artifacts
- Outputs: Review comments/approvals
- Guardrails: Block if `docs/index.html` is missing, broken, or citations unresolved.
- Escalates to: Human

## Publisher (optional)
- Mandate: Confirm GitHub Pages is set to serve from `/docs`.
- Tools: GitHub Pages settings
- Inputs: Approved PR on `main`
- Outputs: Live site at GitHub Pages URL
- Guardrails: No custom domains without explicit approval.
- Escalates to: Human
